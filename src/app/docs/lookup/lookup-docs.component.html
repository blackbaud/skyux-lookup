The lookup text input lets users search a long list of options and make selections.

Use lookup when users must choose an option from a long list of options. It is especially useful when the options are user-defined, but system-defined options can be appropriate as well.

Use lookup fields when users can choose multiple options from a long list.

Use lookup fields when users can add options to the list during their tasks.

Don’t use lookup fields when short lists include 5 options or fewer, especially if it is valuable for users to see all options at once. Use checkboxes checkboxes or radio buttons radio buttons instead.

Don’t use lookup fields when options are highly predictable, especially if they are not unique to the system. Use HTML select fields instead.

Use a single-select to let users select a single option and multi-select to let users select multiple options.

If users may need to add options to the list, include a button to start whatever experience is necessary to create options. After users complete that workflow, the new option is selected in the lookup field.

Use a custom results template when users need more information in the search results to help them select the correct options.

By default, the search and 'Show all/matches' buttons display a native picker with built-in options such as checkboxes for multi-select mode. Use custom pickers instead when different presentations are valuable, such as repeaters repeaters or tree views tree view. This can be combined with a custom results template.

When the lookup component gains focus, the menu appears with buttons for “Show all #” and “New”.

When users enter the minimum number of characters, search results are returned with their text matches highlighted. The “Show all #” button changes to “Show matches (#)”.

At any time, users can click the search button or the “Show all/matches” button to open a modal that lists all available options, or, if search text is entered, all options that match the search text.

When users choose an option in a single-select lookup, the menu or modal closes and the chosen option appears as text inside the text input field.

When users choose an option in a multi-select lookup, the chosen option appears as a token inside the text input field and the cursor returns to the text input field. The menu shows no suggestions until users enter enough characters for another search.

If the lookup field loses focus before users choose a valid option, any text entered is deleted.
